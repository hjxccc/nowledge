# 001 — X 真搜索采用可选 twitter-cli 后端

- 状态：现行
- 日期：2026-07-15
- 来源：`.cairn/tasks/07-15-agent-reach-x-backend/`

## 背景

现有 syndication 数据源免登录且低风险，但只能获取固定账号的缓存时间线，无法覆盖任意关键词的最新讨论。Agent-Reach 维护的 twitter-cli 能提供 Latest 搜索和结构化输出，但需要账号 Cookie，并受 X 的 GraphQL 变化和风控影响。

## 候选

- A. twitter-cli 可用时优先，失败自动退回 syndication（✅ 选中）
- B. 完全替换为 twitter-cli（❌ 否：破坏零配置能力，并把账号风险变成硬依赖）
- C. 保持 syndication，不接真搜索（❌ 否：固定账号覆盖面不足）
- D. 同时内建 OpenCLI 路由（❌ 否：首版复杂度和桌面依赖过高）

## 理由

A 在不改变默认安装条件的前提下补上关键词真搜索；命令存在即使用，任何失败都沿用已经验证的免登录路径。Agent-Reach 仍可在项目外负责安装、升级和健康检查。

## 后果 / 代价

- 真搜索用户需要自行安装并认证 twitter-cli，建议使用专用 X 账号和低频小批量访问。
- X 接口变化时结果会自动降级，学习流程不会失败，但搜索覆盖面会暂时收窄。
- 适配器需要维护 twitter-cli 结构化输出字段的兼容测试。
