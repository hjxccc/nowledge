# Nowledge — 把新主题变成"边做边学"的本地学习包

完整设计见 `DESIGN.md`。核心理念：**别人把料端给你、你读一遍点头就结束（= 认知卸载陷阱）；nowledge 让料真正进脑子。** 产物落 `./<主题>-nowledge/`，零 API Key。

## 三种意图（先分清用户要哪种）

- **快答模式**（最常见入口）：用户只想**快速搞懂/介绍一下**某个东西，尤其是**新鲜度敏感**的（新框架/新工具/"现在 X 怎么样"/"最近火的 X"）。**两条腿一起跑再融合**：① 平台深料——`scripts/ground.py "<主题>" --quick`（context7+HN/掘金 2 源、省聚类、~秒级出前 6 条**带日期新鲜料**）；② 前沿广度——**同时调你自己的原生 `WebSearch` 工具**搜同一主题（这是覆盖极新主题/X·HN·reddit 一手讨论的**主力新鲜源**，不需登录、不打反爬）。把两边结果**当场综述成一段简答**。核心卖点：**AI 训练记忆是旧的（截止 2026-01），快答现查现答、还标出"这点和旧说法不一样"**。
  <important if="主题极新（近几个月发布的模型/框架/工具，如某个新版本号）">此时 `ground.py` 平台源常滞后甚至打不中，**`WebSearch` 必须跑且为主**——别只信 ground.py 的 context7 官方文档，那可能返回泛泛的旧文档而漏掉主题特有信号。</important>
  快答产出契约（必须做到）：① 开门见山 3–6 句答清；② 标 **as-of 日期** + 附 2–3 条现查来源；③ **旧记忆冲突就显式标红**："我训练记忆里会说……但现查是……"（这是最值钱的差异化）；④ 结尾一行升级钩子："想吃透 → 我给你起个学习包（六件套）"。
  🔴 **边界**：只对**新鲜度敏感**的速览触发；纯静态单点事实（"Rust 哪年发布"）仍**直接搜、不走 nowledge**。
- **深学模式**：系统学一个主题 → 走下面第 0 件 + 全套六件套。
- **追踪模式**（保持前沿）：只想看某主题最新动态 → `scripts/track.py "<主题>" --dir <主题>-track`。第 0 件的轻输出：跑一遍选源/采集/判断层，只报**上次之后的新增**（delta），落 `digest.md`（带日期、新在上）。配定时（Cron/ScheduleWakeup）每隔几天自动跑 = 你自己的 last30days，但走 nowledge 判断层、不依赖第三方。
- **发现模式**（快答的特例，"最近有什么值得看"）：不是搜某个已知主题，而是**按时间窗 + star 发现新热点**，如"这月/这周 github 前 N hot 项目"。跑 `scripts/discover.py --since month --limit 5 --out hot.html [--lang python] [--keyword agent]`：github API 现查 `created:>日期 sort:stars`（**非关键词搜、非训练记忆**），出榜单 + 一张**深色榜单卡片 HTML**（模板 `templates/rank-card.html`）。诚实标注：榜单=该窗内**新建**且 star 最高，反映"冒头"不含老牌。
- **三者相通**：快答/发现/追踪里冒出你想吃透的条目 → 直接对它起一个深学包（六件套）。

## 学习模式（深学时的姿势，开场先定，默认「先看懂」，不要一上来就考人）

多数人不喜欢被劈头提问，尤其面对全新概念——这有学习科学撑腰：**新手看完整走查样例 > 硬解题（样例效应 Sweller）**，对纯新手上来"先自答"是"不合意的困难"（挫败≠学习）。所以：

- **A · 先看懂（默认）**：**带读那篇最好的真材料，别现编讲解替代它。** 先甩出第0件选中的**原理源真链接**（大V原文/官方 quickstart/arXiv）让用户读，nowledge 只加**大V没给的三件增值**：① 对齐用户水平/语言的重点与生词；② 这篇没讲但会踩的坑（拉踩坑源）；③ 该记住的 3 个点。读完只做**一次轻量回顾**（可跳过），不懂处记成"洞"进复训队列。全新概念/新手一律走这个。
  > ⚠️ nowledge 的价值不在"讲得比大V好"（比不过、还会 slop），在**给最好的教程加一个"真进脑子"的外壳**：三角交叉 + 后半场提取复训。所以"先看懂"**必须锚定真源**，不许从训练记忆现编一篇替代原文。
- **B · 先自答（挑战型，opt-in）**：先抛问题让用户猜/写，再看答案。仅当用户**有底子或主动要挑战**时用。
- **C · 混合**：看懂一个样例 → 自己仿一个（渐隐样例/补全）。

开场一句话定模式（"想我先带你看懂，还是你先自己试试？"），或对全新主题**默认 A**。护栏没松：**看在前、回在后**——先看懂之后，仍至少来一次轻提取（讲给 AI 听 / 一道回顾题）+ 间隔复训照常，只是不再拿"先自答"当强制门槛。

## 主流程（第 0 件接地 → 六件套）

🔴 **CHECKPOINT · 落点确认（建目录树前必做）**：深学/发现会创建 `./<主题>-nowledge/` 一整棵目录。**动手前先明说落点并确认**——"我会建在 `<当前目录绝对路径>/<主题>-nowledge/`，可以吗？还是换个位置？"。别默默落在 agent 的 cwd（用户常没料到 cwd 在哪，一落一大堆文件）。追踪模式的 `--dir` 同理先确认。

**第 0 件 · 接地引擎**（先找料，不信 AI 训练记忆）
1. 按 (主题域 × 语言偏好 × 广度/深度) 选 2–4 个源，见 `references/source-registry.md`。一期内建 T0 免费层 + 中文公众号/掘金。
2. 采集 → 去重 → 聚类选代表 → 加权排序 → 学习价值打分。算法见 `references/judgment-pipeline.md`。
3. 每个知识点凑齐**角色三角**：原理源(官方/arXiv) + 实操源(掘金/quickstart) + 踩坑源(HN/知乎)。缺哪补哪。
4. 产出 `sources/` 料清单（带来源+日期+权威度+角色标签）。
   🔴 **CHECKPOINT · 护栏**：料清单必须显式标注"这是社区在说什么，**不是答案**"，并 **STOP** 先让用户/AI 基于料自己提炼路线，再进六件套。不许直接把料当结论喂下去。

**六件套**（喂料，逐件产出）
- ① `ROADMAP.md` — 只留关键 20%，**慢变量(原理)/快变量(选型)分层**，快变量标 `as-of` 日期+厂商偏向。模板 `templates/ROADMAP.md`。
  ⚖️ **知识与接地怎么配**：**慢变量(原理/经典主题)骨架，用模型既有知识搭得更成体系**——接地源负责**引用、验证、补盲、纠偏**，别只堆散料出个薄骨架；**快变量(选型/新东西)一律以联网现查为准、不信训练记忆**。即：老主题模型搭台、料校准；新主题料说了算。别自废模型武功，也别让模型裸奔。
- ② `diagrams/*.html` — 复杂机制做成图解，逐条过 `references/mayer-checklist.md`。喂料 = 原理源。**先判类型再选模板**：
  - **"调一个量看结果变"型**（超参数影响、收敛/发散、注意力权重、概率分布、算法单步执行……）→ 出**可拖动交互图**（canvas+滑块），模板 `templates/diagram-interactive.html`，换主题只改 4 处 ★（model/classify/draw/控件）。**主动操纵 > 被动看图**（学习科学）——这类概念别用静态图糊弄。
  - **其余静态结构/流程**（架构、数据流、时序）→ 深色 SVG，模板 `templates/diagram-dark.html`。
- ③ `exercises/` — 15 分钟能跑的最小 MVP，从基础版手写到进阶版。喂料 = 实操源。
- ④ `checkpoints/` — **按模式走**：模式 A 先给讲解+样例、看懂后做一次**轻量回顾**（可跳）；模式 B 才**先合上材料自答再看答案**。答不上的记成"洞"入⑤。模板 `templates/checkpoint.md`。喂料 = 踩坑源。
- ⑤ 间隔复训 — checkpoint 的"洞"入队 `progress.json`，由 `scripts/review.py` 真调度（`add`/`due`/`grade`/`stats`，间隔 1/3/7/16/35 天，答对拉长答错归零）。`REVISIT.md` 写用法。**这一件是别人没有的差异化，不是让用户自觉。**
- ⑥ `checkpoints/teach-*.md` — 用大白话讲给 AI 听，**AI 扮学生只追问、指出讲不清处，不代讲**。

贯穿：**合意困难**(别做太顺)、**信息缺口驱动**(先抛一个用户完不成的真实项目制造张力)。

## 硬规则（防认知卸载，必须遵守，别退化成答案机）

1. **看在前、回在后**：默认「先看懂」，别劈头考人；但绝不能纯被动读到底——看懂后**至少一次轻提取**（讲给 AI 听 / 一道回顾题）+ 间隔复训。只有模式 B 才"先自答后看答案"。
2. "讲给 AI 听"AI **只追问不代讲**。
3. 快变量一律**联网现查 + 标日期 + 标厂商偏向**，不信训练记忆。
4. 第 0 件产出是**原料不是结论**；诚实交代**未核实项**。
5. **最小主义**：路线图只留关键 20%，每个知识点每角色只留 1 篇代表，不堆料。

## 失败模式（第 0 件采集常见坑，按此降级，别静默跳过）

| 触发条件 | 一线修复 | 仍失败兜底 |
|---|---|---|
| 某源 403/被封（如 Reddit）| `ground.py` 已 try 捕获并优雅跳过该源 | 该角色缺料时换同角色其他源（踩坑源 Reddit→HN/知乎）|
| 掘金/搜狗反爬返回空 | wechat.py 两层取数：curl 被 antispider→**自动走 CDP 代理**(localhost:3456,带会话过反爬) | CDP 也没起/仍需过验证码→优雅返回空；正文走站点经验 `mp.weixin.qq.com.md` |
| arXiv 返回一堆不相关论文 | 说明主题是实践术语，planner 应判 `ai-tooling` 剔除 arXiv | 手动 `--exclude arxiv` 或 `--domain ai-tooling` 重跑 |
| 缺**原理**源（zh 主题 arxiv 被剔常见）| `ground.py` 自动调 context7 拉官方文档兜底（缺 principle 才触发）| `--sources` 手动加 context7 |
| 缺**踩坑**源（zh 主题，知乎需登录常空）| 掘金/微信实操文里带"避坑/血泪/坏习惯"的自动 retag 成踩坑源（`looks_like_pitfall`）| 仍空则 ROADMAP 诚实标"该角色缺料"，不编造；知乎登录后可白捡 |
| 全部源都空/无网 | 退到 WebSearch 兜底 | 告知用户联网受限，改用其手动提供的材料 |
| 快变量信息与 AI 记忆冲突 | **一律信联网现查的、标日期的**，不信训练记忆 | 标注为未核实，让用户定夺 |
