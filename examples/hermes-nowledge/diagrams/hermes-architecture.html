Hermes Agent · 框架原理

核心 = 一个 run_conversation() 循环：调模型 → 有工具调用就执行再回炉 → 纯文本才收工； 外挂持久记忆与技能系统，靠上下文压缩撑住长跑。 （据官方 agent/conversation_loop.py · agent-loop.md，非训练记忆）
一句话看图：中间黄框是心跳——模型每轮要么"调工具再回炉"（红，agent 干活），要么"出纯文本收工"（青）。 左边青色 = 它凭什么跨会话不忘（外置记忆）；右边紫色 = 它凭什么越用越强（技能→自我进化）； 循环底部红色虚线 = 它凭什么能长跑不爆（上下文压力阀）。三样都去掉，就退化成一个普通一次性问答。

术语速查

agent loop：调模型→执行工具→回炉的循环，agent 的心跳
tool_calls：模型要求执行的工具调用；有它就继续循环
持久记忆：落磁盘、跨会话的精炼笔记（有上限）
procedural memory：把动作序列固化成可复用技能
preflight 压缩：上下文超 50% 时先压缩再继续
provider 适配：同循环兼容 OpenAI/Codex/Anthropic
