# 判断管线（"结合哪几个"的确定答案）

> 核心信条（抄卡兹克 11 版血泪）：**能用代码就别用模型。** 模型只做"打分 + 写一句为什么选它"；
> 选源/去重/聚类/角色配齐/排序全用**确定性代码**。
> 算法参考 last30days-skill（MIT, © 2026 Matt Van Horn）**自包含重写**，非 vendoring。

## 管线

```
planner → fanout → dedupe → cluster(MMR) → fusion(RRF) → score(学习价值)
```

### 1. dedupe — URL 归一化去重
- 小写、剥 `www./old./m.` 前缀、去 `utm_*/ref/fbclid` 等 tracking 参数、去锚点。
- 归一化后 URL 相同 → 判重，保权威度高的那条。

### 2. cluster — 实体聚类 + MMR 选代表
- **实体提取**：从标题+摘要抽专有名词/大写词/数字（不用 embedding，够用且零依赖）。
- **聚类**：实体重叠率 > 阈值归为一簇（同一主题的多篇）。
- **MMR 选代表**（Carbonell & Goldstein 1998）：每簇里按
  `score = λ·final_score − (1−λ)·与已选的相似度` 迭代选，λ≈0.7。
  → 既选高质量、又保证彼此不重复（解决"十篇都在讲同一件事"）。

### 3. fusion — 加权 RRF 融合排序
- **RRF**（Cormack et al. 2009，`RRF_K=60`）：多个源各自的排序列表，按
  `Σ  weight / (RRF_K + rank)` 融合成一条总排序。
- `weight = source_weight（见 source-registry） × 子查询权重`。
  → T0 官方源天然压过 T2 二手转述。
- 加 **per-author cap**：同一作者/号最多进 N 条，防刷屏。

### 4. score — 学习价值打分（换掉 last30days 的新闻维度）

| 维度 | 打分依据 | 谁来打 |
|---|---|---|
| 覆盖度 | 命中 ROADMAP 哪个模块（避免重复堆同一块） | 代码规则 + 模型判类 |
| 难度匹配 | 入门/进阶，对齐当前 checkpoint（合意困难：略高于现有） | 模型打分 |
| 权威度 | = source_weight（一手 > 二手） | 代码 |
| 时效 | 快变量要新且有日期；慢变量原理老的也行 | 代码 |

## "结合哪几个" = MMR 多样性 + 角色三角

MMR 只保证不重复，**还要保证互补齐全**。每个 ROADMAP 模块凑齐三个角色，缺哪补哪：

```
知识点 = 原理源×1 (官方文档/arXiv，是什么·为什么)  → 喂 图解②
       + 实操源×1 (掘金/官方 quickstart，15分钟能跑) → 喂 练习③
       + 踩坑源×1 (HN/知乎，真实困惑与分歧)          → 喂 追问④
```
每篇挂一句**推荐理由**（判断，不是再复述摘要）："选它因为它补的是【实操】这一块。"

## 模型 / 代码分工红线

- ✅ 模型：给单条打分、写推荐理由、判断内容归哪个模块。
- ✅ 代码：选源、去重、聚类、MMR、RRF、角色配齐、阈值。
- ❌ 别把"选哪几篇/最终排序"整个交给模型 —— 卡兹克 V7→V8 就是这么负优化的。
